Introduction

Thide guide will cover the new (as of v1.11) "Labels" system in Simple Planes. Labels come in two forms: one as the standalone Label part, and the other as a dedicated built-in label on the Switch and Button parts to (optionally) label their function. Regardless of which one you're using, they all are provided through the TextMesh Pro addon for the Unity game engine (which is what powers Simple Planes).

TextMesh Pro is, on the surface, a means of offering placable 2D Text in the game environment (i.e. Labels), then also provides a lot of "Rich Text" support to format your text. It offers everything from simple things like bold, italics, and underline; as well as the ability to change fonts, apply styling, curve text, individually adjust character placement, and more. That, combined with what Simple Planes already offers, also means that these Labels allow you to place Funky Trees code to be displayed, converting your Label from a once-static text box into a dynamic text-based display!

Those familiar with HTML/CSS coding, or even older web forums, will already be quite familiar working with these Labels. Rich Text is not too far removed from HTML, and before there was support for shortcuts like Ctrl+B to make your forum post text bold, you had to manually type out the formatting codes in this way. This is similar to the "Markdown" that the Simple Planes website supports, but much easier to understand... albeit at the expense of there being more text (code) to look at.

Here's how TextMesh Pro explains it in the documentation:

You can use rich text tags to alter the appearance and layout of your text. These tags work like HTML or XML tags, but have less strict syntax.

A tag looks like <tag>. Many tags operate on a scope, which you can end with </tag>. Such scopes can be nested, and you don't have to close them in the same order that you started them.

Some tags have values and attributes, like <tag=value> and <tag attribute=value>. These arguments are either names or numeric values. Numbers are either regular decimal numbers, pixels like 1px, percentages like 80%, font units like 1.2em, or hexadecimal color values like #FF. Names can be either with or without double quotes, but if there are more attributes, it's best to use quotes.

Tags, plus their attributes, can be up to 128 characters total. This limitation shouldn't be an issue, unless you're using very long string attributes.

NOTE: As we'll explore in the guide, there are a couple exceptions to certain parts of what they stated.
