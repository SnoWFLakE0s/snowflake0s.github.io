2. Using the Input Dialog / Debugger

docs / tutorial / console /

Written by SnoWFLakE0s

As it would most certainly be beneficial to know where to even write Funky Trees code, this section will walk you through the various means of accessing the places that Funky Trees code can be written. There largely are two types of interfaces you can use: the input interface, and the console debugger. The input dialog is used to actually execute and control a part with your code, while the debugger allows you to very easily see and experiment with Funky Trees code without actually having to have any physical parts.

A. Using the input Dialog

The input property is an attribute that controls the behavior of the vast majority of dynamic SimplePlanes parts. As an XML property, it can be accessed through three standard ways:

  • Part properties menu
  • Overload editor mod
  • Direct XML editing

Any of these three methods are reliable and will give you the same results, but with differing degrees of ease-of-use. The part properties menu is the easiest to access (and is also the newest method of access on this list). Simply select the part you want to program, and click the part properties icon:

This will bring up a sidebar menu, from which you can find the inputController dialog, which will look like the below: [Show image]

Now simply Ctrl + Click on the input field to open up the input property editor, to which you can write Funky Trees code. [Show image]

If you wish to utilize the Overload XML editor mod, you can do so by simply opening SimplePlanes, pressing the MODS tab, and enabling OVERLOAD. You can then enter the designer, from which you will be able to see the Overload icon:

Now select the part you wish to program, click the open the menu and select the inputController group. The input attribute should be visible. You can insert/write your code into the value field and click the check button to save your changes.

As this tutorial is not meant to be a general XML editing tutorial, the procedure for editing raw aircraft XML files will not be covered. However, the procedure remains identical: find the part you wish to program, find the inputController group of attributes, and edit the value of the input property. Any general purpose text editor program can be used for this task.

Regardless of the method used to input code into a part, once it is saved with code, SimplePlanes will attempt to control the part according to the code inside as soon as the level is loaded. There is no manual execution needed to "start" code execution within a component.

To see for yourself how the input dialog works, use any of the methods above to create a rotator with the input of Time. Once loaded in the level, you will be able to see the rotator continually rotates.

B. Using the Console Debugger

The console debugger is a little display item you can generate via the DebugExpression command. It is well-suited for testing new Funky Trees code, testing parts of existing code, or even doing some basic telemetry without setting up other measuring devices.

To use the console debugger, you must first open the console, which you can do by pressing the tilde key [`] on Windows/macOS. For mobile platforms, you must first enable the console by going to the Settings tab in the main menu, and tapping the word SETTINGS 5 times. Then, you can open the console at any time by tapping the screen with three fingers. Whichever method is used, opening the console will bring up a input field that accepts text inputs. Type the command DebugExpression, add a space, and open with a double quotation. Now write any Funky Trees code you wish to test, and close with another double quotation. Finally, press enter to execute the console command.

As an example, let's try the expression Time in order to check the elapsed simulation time. As with standard written programming notation, a command is indicated by three preceding greater-than signs: >>>, and the results of the command is displayed on the next line.

  • >>> DebugExpression "Time"
    Time: 0.000

If you did all of this properly, you should be able to see a small window in the top-left corner of the screen with the expression and the value of the expression. Note, this window is only visible while in the sandbox and not in the designer or otherwise. The console debugger is incredibly useful as it not only allows you to easily check that your code is written to standard, it also allows you to directly view the value your expression will evaluate to.

By looking at the actual value of the code, it becomes clear why a rotator with this code will continually rotate--the value of Time simply keeps increasing. When you build more complex code, for example if we were to do some math with our Time value, the console debugger can even be used to break down a larger expression into subcomponents to get a better idea of what each part of the entire expression is doing.

For this example, let's try adding 4 to our original Time value, then dividing Time by this new added value. That will give us the expression (Time)/(Time + 4). We will use the console debugger on each part of the division to see what each is doing, and the actual full expression.

  • >>> DebugExpression "(Time)"
    (Time): 0.000
  • >>> DebugExpression "(Time + 4)"
    (Time + 4): 0.000
  • >>> DebugExpression "(Time)/(Time + 4)"
    (Time)/(Time + 4): 0.000

Throughout this tutorial the proper usage of both input and the console debugger is recommended. Now you are well equipped to not only try the various examples available, but also actually begin programming in Funky Trees!
