5. Operators in Funky Trees

docs / tutorial / operators /

Written by SnoWFLakE0s

In the previous section, we covered the topic of data types. Data sounds pretty useful, but data in itself can't quite do much. Just because we know exactly what we want to do, doesn't mean we actually did anything. Operators are a fancy term for certain mathematical & programmatical actions you perform with pieces of data in order to acheive a desired result. In this section of the tutorial, we will cover the two types of operators in Funky Trees.

A. Math Operators

The first type of operator we will discuss are math operators. Math operators are conceptually very simple, and will be obvious to anyone who's been through basic algebra in school: they perform basic algebraic calculations. The following table lists all available math operators, their function, general forms, and basic examples. Assume that a, b, and c are other expressions or constants[?].

While spaces around operator symbols are not required, sometimes no spacing can cause errors in how your expression is evaluated. For the sake of consistency, it is recommended you put spaces around an operator.

Symbol Usage General Form Explanation Examples
+ Addition a + b Yields the value a + b. Additionally, strings can be concatenated[?] through the addition operator. For example, the statement "He" + "llo" will yield the resulting string "Hello".
  • >>> DebugExpression "1 + 1"
    1 + 1: 2
- Subtraction, Negative sign a - b
or -c		 Yields the value a - b, or the value -c. Take caution to ensure that -c is written when needing the negative of c, as in a real-world environment writing - c can result in subtracting c from what precedes it.
  • >>> DebugExpression "2 - 1"
    2 - 1: 1
* Multiplication a * b Yields the value a * b.
  • >>> DebugExpression "10 * 10"
    10 * 10: 100
/ Division a / b Yields the value a / b.
  • >>> DebugExpression "6 / 3"
    6 / 3: 2
() Parentheses (group) (a + b) * c Groups statements. In this example, the operation a + b is done before c is multiplied. Note: implied multiplication, like in the form of (a)(b), is not supported. To do this, you must use the multiplication operator, that is, (a)*(b).
  • >>> DebugExpression "7 - 1 / 3"
    7 - 1 / 3: 6.666666666
  • >>> DebugExpression "(7 - 1) / 3"
    (7 - 1) / 3: 2

The whole point of math operators is to enable you to figure out how to take data and manipulate it in a way that is useful to you. For example, the input variable Time tells you the number of seconds that have passed since the sandbox was loaded. If you wanted to find the number of minutes that have passed instead, this is obvious to anyone who knows how to tell the time--simply divide by 60. We can do exactly that:

  • >>> DebugExpression "Time"
    Time: 0.000
  • >>> DebugExpression "Time / 60"
    Time / 60: 0.000

Math operators are really the basis upon which practically any sort of Funky Trees code is built on. You can apply them in countless ways to do really anything in SimplePlanes, so it is important to understand how you can use them.


B. Logic Operators

The second type of operators are called logic operators. They may seem confusing at first, especially for those just learning how to code, as they rely on something called boolean logic instead of simple algebra. There is no need to worry, they are as simple as math operators once you understand how they work.

The result of doing a logic operation is always one of two things. They always output something known as a boolean value as the ouput, either True or False. There are three types of logic operators in Funky Trees: comparison, boolean, and selection operators. This section will cover how all three of them work, and how you may use them to code in Funky Trees.


  • 1. Comparison Operators

    Comparison operators perform comparison operations.

    All comparison operators do is compare two values and tell you something about the relation between those values. Comparison operators only take numbers as inputs, and only output booleans as outputs. You may have already seen these symbols back in school:

    4 < 6 is True.
    4 > 6 is False.
    1 = 1 is True.

    This is all there is to it, really. What makes this a little more complicated is if you start adding extra things, like the math operators from earlier:

    1 + 1 = 2 is True.
    1 + 1 < 0 is False.

    Funky Trees is really about not how you use these operators one-on-one, but how you take these basic building blocks to create a longer piece of code that will perform an advanced, higher level functions not possible by other "normal" means. The table below lists all the comparison operators available to you in Funky Trees and how to use them.

    Symbol Usage General Form Explanation Examples
    = Equality a = b Checks if a = b. If a is equal to b, the output will be True. In any other case, it will be False.
    • >>> DebugExpression "(2 + 2) = 4"
      (2 + 2) = 4: True
    • >>> DebugExpression "4 = 2"
      4 = 2: False
    != Nonequality (NOT) a != b Checks if a != b. If a is NOT equal to b, the output will be True. In any other case, it will be False.
    • >>> DebugExpression "4 != 2"
      4 != 2: True
    • >>> DebugExpression "(2 + 2) != 4"
      (2 + 2) != 4: False
    > Greater than a > b Checks if a > b. If a is greater than b, the output will be True. In any other case, it will be False.
    • >>> DebugExpression "2 > 1"
      2 > 1: True
    • >>> DebugExpression "(2 + 1) > 4"
      (2 + 1) > 4: False
    < Less than a < b Checks if a < b. If a is less than b, the output will be True. In any other case, it will be False.
    • >>> DebugExpression "1 < 2"
      1 < 2: True
    • >>> DebugExpression "4 < (1 + 1)"
      4 < (1 + 1): False
    >= Greater than or equal to a >= b Checks if a >= b. If a is greater than, or equal to b, the output will be True. In any other case, it will be False.
    • >>> DebugExpression "3 >= 2"
      3 >= 2: True
    • >>> DebugExpression "2 >= 2"
      2 >= 2: True
    • >>> DebugExpression "(2 - 1) >= 2"
      (2 - 1) >= 2: False
    <= Less than or equal to a <= b Checks if a <= b. If a is less than, or equal to b, the output will be True. In any other case, it will be False.
    • >>> DebugExpression "1 <= 10"
      1 <= 10: True
    • >>> DebugExpression "(5 + 5) <= 10"
      (5 + 5) <= 10: True
    • >>> DebugExpression "(2 + 1) <= (2 * 1)"
      (2 + 1) <= (2 * 1): False

    As some of the examples above show, it is possible to combine usage of the math operators discussed earlier in the section with comparison operators to perform more advanced arithmetic. Let's say we want to write some code that allows us to figure out if 5238 times 23 subtracted by 42069 is larger than 6969. How would we approach this problem?

    First, you must use math operators to do the math before we see if that number we calculate is larger than 6969. This is easy: just take 5238 and multiply by 23:
    5239 * 23

    We then subtract 42069:
    5239 * 23 - 42069

    Then finally, we group this statement to ensure Funky Trees isn't confused about what number it's trying to compare to 6969:
    (5239 * 23 - 42069)

    All we now do is compare:
    (5239 * 23 - 42069) > 6969

    • >>> DebugExpression "(5239 * 23 - 42069) > 6969"
      (5239 * 23 - 42069) > 6969: True

    Congratulations! You just successfully calculated a result and performed a logic operation on it to learn a new fact.


  • 2. Boolean Operators

    Boolean operators perform boolean logic operations.

    Boolean operators will compare boolean values[?] and tell you something about the relationship between those boolean values. Boolean operators will only take booleans as inputs, and only output booleans as outputs. The word boolean may throw new programmers off, but once you understand that booleans are literally only either True or False this makes them one of the most simple kinds of data to operate with.

    Symbol Usage General Form Explanation Examples
    & Boolean AND a & b Checks if both a and b are True. If both operands aren't True, the result will be False, that is, if any one of the two operands are False, the expression will output False.
    • >>> DebugExpression "True & True"
      True & True: True
    • >>> DebugExpression "True & False"
      True & False: False
    • >>> DebugExpression "False & False"
      False & False: False
    | Boolean OR a | b Checks if either one of a and b are True. Only one of the two operands need to be True for the operation to output True. This operator only ouputs False if both operands are False.
    • >>> DebugExpression "True | True"
      True | True: True
    • >>> DebugExpression "True | False"
      True | False: False
    • >>> DebugExpression "False | False"
      False | False: False
    ! Boolean Invert !a This operator inverts the value of a boolean. That is, !True is False, and !False is True.
    • >>> DebugExpression "!True"
      !True: False
    • >>> DebugExpression "!False"
      !False: True

    Say you want to compare two statements and ensure both are true. A common example would be finding a timeframe, like figuring out if we are at least a minute into entering the sandbox, but also under 30 minutes since entering the sandbox. How do you approach this problem?

    We can reuse the Time example from earlier in the section to first figure out the number of minutes that have passed since starting the sandbox:
    Time / 60

    Now figure out if the number of minutes have passed at least one minute using comparison operators:
    (Time / 60) >= 1

    Do the same, but for under 30 minutes:
    (Time / 60) < 30

    For the sandbox uptime to be both above at least 1 minute and under 30 minutes, both of the code snippets we just wrote need to be True. The AND boolean operator is perfect for this task, and we arrange the snippets to check if both are True:
    ((Time / 60) >= 1) & ((Time / 60) < 30)

    • >>> DebugExpression "Time / 60"
      Time / 60: 0.000
    • >>> DebugExpression "((Time / 60) >= 1) & ((Time / 60) < 30)"
      False

    Congratulations! This is actually a fairly common type of code that is used for a variety of applications for many SimplePlanes creations.


  • 3. Selection Operators

    Selection operators output one of the two expressions as output depending on the value of the initial condition.

    Selection operators are one of a kind in Funky Trees. They are versatile because they allow you to switch things on and off using the conditional operand. To put this into context, every other logic operator can only output booleans as you saw above; selection operators can do a lot more than that, as you will see below.

    Symbol Usage General Form Explanation Examples
    ? : IF (ternary selection) a ? b : c Checks if a is True or False. If a is True, the operator outputs the value b. If a is False, the operator outputs the value c. This also means the selection operator can have a variety of datatypes for outputs (but a must be a boolean value).
    • >>> DebugExpression "True ? 1 : 2"
      True ? 1 : 2: 1
    • >>> DebugExpression "False ? 1 : #This is a String#"
      False ? 1 : "This is a String": This is a String

    We can use the selection operator to spice up the timeframe example from earlier. Instead of just reading True or False, let's have it read out a sentence depending on the time. The code we wrote earlier already outputs a boolean as a result, so all we need to do is use it as the boolean condition for the selection operator. Let's write a couple strings that will say "Time is (outside/within) 1 to 30 minutes":
    ((Time / 60) >= 1) & ((Time / 60) < 30) ? "Time is within 1 to 30 minutes" : "Time is outside 1 to 30 minutes"

    In the console debugger:

    • >>> DebugExpression "((Time / 60) >= 1) & ((Time / 60) < 30) ? #Time is within 1 to 30 minutes# : #Time is outside 1 to 30 minutes#"
      ((Time / 60) >= 1) & ((Time / 60) < 30) ? "Time is within 1 to 30 minutes": "Time is outside 1 to 30 minutes"

    If you were unable to witness this code in action, try reloading the page and wait a minute.
