Introduction

This guide will cover the new (as of v1.11) "Labels" system in Simple Planes. Labels come in two forms: one as the standalone Label part, and the other as a dedicated built-in label on the Switch and Button parts to (optionally) label their function. Regardless of which one you're using, they all are provided through the TextMesh Pro add-on for the Unity game engine (which is what powers Simple Planes).

TextMesh Pro is, on the surface, a means of offering placeable 2D Text in the game environment (i.e. Labels), then also provides a lot of "Rich Text" support to format your text. It offers everything from simple things like bold, italics, and underline; as well as the ability to change fonts, apply styling, curve text, individually adjust character placement, and more. That, combined with what Simple Planes already offers, also means that these Labels allow you to place Funky Trees code to be displayed, converting your Label from a once-static text box into a dynamic text-based display!

Those familiar with HTML/CSS coding, or even older web forums, will already be quite familiar working with these Labels. Rich Text is not too far removed from HTML, and before there was support for shortcuts like Ctrl+B to make your forum post text bold, you had to manually type out the formatting codes in this way. This is similar to the "Markdown" that the Simple Planes website supports, but much easier to understand... albeit at the expense of there being more text (code) to look at.

Here's how the official TextMesh Pro documentation explains it

You can use rich text tags to alter the appearance and layout of your text. These tags work like HTML or XML tags, but have less strict syntax.

A tag looks like <tag>. Many tags operate on a scope, which you can end with </tag>. Such scopes can be nested, and you don't have to close them in the same order that you started them.

Some tags have values and attributes, like <tag=value> and <tag attribute=value>. These arguments are either names or numeric values. Numbers are either regular decimal numbers, pixels like 1px, percentages like 80%, font units like 1.2em, or hexadecimal color values like #FF. Names can be either with or without double quotes, but if there are more attributes, it's best to use quotes.

Tags, plus their attributes, can be up to 128 characters total. This limitation shouldn't be an issue, unless you're using very long string attributes.

NOTE: As we'll explore in the guide, there are a couple exceptions to certain parts of what they stated.


RichText Coding

This section will primarily serve to provide you with a rundown of what text formatting you have access to. The majority of it will be no different than the previously mentioned official documentation; however, most sections will include further details or tips for its usage.

Currently, there are a few RichText codes that either do not work in SimplePlanes because of a limitation (or Jundroo's choice to disable it), or due to a lack of understanding on using the code. Over time, as users explore and understand the capabilities, these questions will get answered more definitively. At this moment, any suffixed with an Asterisk (*) are those whose functionality is unknown; therefore, may actually not work in Simple Planes.

A Quick Word About Syntax
These are specific to RichText, albeit similar to that in the Funky Trees Guide; however, when it comes to the inclusion of actual FT code into the Labels, you will need to abide by those Syntax rules as well! They do not cross-over at any point, so when you "exit" the RichText domain and "enter" that Funky Trees domain, that is where the Syntax varies.

That being said:

  • Spelling matters, including spaces and hyphens.
  • Case Sensativity: to make life simple, only use <lowercase> within tags (there's an exception, which is really not worth the hassle, but the tag word works as all-uppercase; also the value for <font=> allows mixed-cases but as that tag doesn't work it's irrelevant).
  • Tag and Value Separator: the Equal (=) sign is the only way, a Colon (:) is not allowed; <tag=value> will work, <tag:value> will not.
  • One small error leads to the entire <tag> failing and all associated text not being parsed. Thankfully, this error is easy to spot and easy to address, because everything included with the tag will be displayed in your label. Initially it may be using a Colon in place of the Equals. Perhaps you wrapped your value in Quotes when that tag does not accept them. Or, forgotten the quotes when necessary.

Breakdown of Value Types and Modifiers:
<tag=valuemodifier>

A general tag is formatted as such and the value filled in using Numbers, Words, or {funky expressions};
numbers are whole numbers (1, 2, ..., n) or decimals (1.234), and positive or negative (- preffixed).
<tag=#%|em|e>
A tag that accepts Numbers, and any optional modifiers it supports ("em" and "e" do the same thing).
If no modifier is specified then the number is treated as a Multiplier. Any unsupported text or characters
included with a number will be completely ignored (skipped over); EXAMPLE: entering "1,3" instead
of "1.3" -- an accidental comma instead of a decimal -- will therefore be treated as "13".
<tag=option1|option2|...|optionn>
A tag that accepts Words, and all the various|choices|...|accepted by that tag; typed as
all-lowercase unless otherwise specified.
