3. General Rules for Writing Code

docs / tutorial / syntax /

Written by SnoWFLakE0s

Programming is a fairly precise operation, and there are certain rules that must be adhered to if code is to behave in the manner intended by the programmer. If you are new to coding, this section may help you understand some general rules for writing Funky Trees code (which generally apply to other coding languages as well). If you have some background with coding in general, this may be obvious information. That being said, it is advised that the section at the very least be skimmed.

A. Case and Spelling Sensitivity

Careful attention must be paid to the exact spelling and notation of code. As with virtually any programming language, the Funky Trees language is also fully case and spelling sensitive, and as such any minute spelling error can cause code to fail to evaluate, evaluate to the wrong value, or even cause corruption-inducing errors.

Continuing the example of using the Time data variable from the earlier section, when used in the debugger Time will evaluate in the expected manner, as seen below:

  • >>> DebugExpression "Time"
    Time: 0.000

Any misspellings or capitalization errors will fail in returning any value, with the error message shown below. If this is forced onto a part input, the console will return an error message with the form: Invalid expression on part [partId], where [partId] will be the part ID of the part in which the invalid expression has been entered.

  • >>> DebugExpression "time"
    Name not defined: time
  • >>> DebugExpression "Tim"
    Name not defined: Tim
B. Order of Operations

Mathematical operations will be very frequently utilized in Funky Trees programming. It is imperative that care be taken to how math operations are structured and written. A small mistake in writing a formula can result in a completely different value, leading to code not working in the desired way.

The most core order of operations will be the rule commonly known as PEMDAS. That is, math will be evaluated in the order of expressions inside parentheses, exponentials, multiplication, division, addition, and finally subtraction. Some basic examples are presented below.

  • >>> DebugExpression "Time / 4 + 1"
    Time / 4 + 1: 0.000
  • >>> DebugExpression "Time / (4 + 1)"
    Time / (4 + 1): 0.000

Notice that the grouping of parentheses changes the order of operations (and therefore the final result).

These rules are not fully exhaustive. For example, there are certain quirks with how certain special operations are handled, but these are not relevant to the general use case of a new Funky Trees programmer. For more in-depth information, the reference documentation should be consulted.
